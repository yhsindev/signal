# Signal 訊源

> 不追報導，回到源頭。

---

## 專案理念

資訊時代不缺資訊，缺的是判斷。

每天湧入的科技新聞裡，大多數內容是同一件事的不同包裝——從一份財報、一個工程 Blog、一場法說會，輾轉被翻譯、摘要、再摘要，等到抵達你面前時，決策者的原始邏輯早已消失在層層轉述裡。

Signal 訊源從另一個方向出發：直接讀第一手資料，記錄自己的判斷，追蹤真正值得長期關注的主題。它不是聚合器，不是 RSS reader，而是一個讓你沉澱思考的個人情報工作台。

這個工具背後的問題意識，來自一個簡單卻很少被正面回答的問題：

**「作為一個寫程式的人，我在這場產業變動裡站在哪裡？」**

---

## 名稱由來

### Signal

通訊工程裡，signal 和 noise 永遠同時存在。接收器的任務不是消滅雜訊，而是從雜訊中識別出真正的訊號。這個比喻精確地描述了今天閱讀科技資訊的處境：內容愈來愈多，真正重要的那條線卻愈來愈難找。

Signal，就是那條值得你花時間追蹤的線。

### 訊源

中文名取「訊息的源頭」之意。「訊」對應 signal，「源」對應 source——回到第一手，不繞路。兩個字放在一起，也隱含一種工作態度：找到訊息從哪裡來，而不只是知道它說了什麼。

---

## 目標

Signal 訊源有三個具體目標，依序遞進：

**1. 建立高品質的閱讀習慣**

不是「每天看幾篇」，而是「每篇都留下自己的判斷」。每筆閱讀記錄都必須填寫個人觀點欄位，沒有自己的詮釋，閱讀狀態就不算「消化完畢」。

**2. 追蹤產業主題，而不是單篇文章**

Nvidia 的記憶體分工、三大雲端的商業模式差異、台灣在全球供應鏈的位置——這些主題橫跨財報、技術文件、政策報告，無法從單一來源理解。Signal 讓你把散落各處的閱讀聚合到同一個主題脈絡下，逐步建立完整的判斷框架。

**3. 釐清自己作為工程師的位置**

不是技能清單，是價值鏈分析。底層系統、AI 基礎設施、應用產品、半導體生態——每個層次的進入門檻、長期需求、與 AI 浪潮的交互方式都不同。Signal 的「工程師定位」模組，幫你把閱讀到的產業趨勢，對應到自己正在走的那條路上。

## 技術架構

```
signal/
├── src/
│   ├── components/       # UI 元件
│   ├── pages/
│   │   ├── Reading/      # 閱讀清單 + Claude 摘要
│   │   ├── Topics/       # 主題追蹤地圖
│   │   ├── Sources/      # 來源管理
│   │   └── Position/     # 工程師定位分析
│   ├── stores/           # Zustand 狀態管理
│   ├── services/
│   │   └── claude.ts     # Claude API 摘要服務
│   └── data/
│       └── sources.json  # 預設來源清單
├── README.md
└── package.json
```

**Tech Stack：** React + Vite + TypeScript + Tailwind CSS + Zustand + Claude API

所有資料存於 localStorage，不需要後端服務。Claude API 用於閱讀摘要，需自行設定 API Key。

## 設計原則

- **嚴肅的研究工具，不是資訊娛樂介面。** 視覺設計偏向學術期刊感，而非科技媒體感。
- **閱讀不等於消化。** 沒有自己觀點的文章，狀態永遠是「待消化」。
- **第一手優先。** 能讀原文就不讀摘要，能看財報就不看分析師報告。
- **主題重於文章。** 真正有價值的理解，來自同一主題的多篇文章在時間軸上的累積。
